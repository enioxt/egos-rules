+# Donation Gateway Plan — Sustentação Comunitária (Sem Paywall/Premium)
+
+> **Status:** Proposed (v2)
+> **Date:** 2026-03-03
+> **Principle:** acesso ao dado permanece aberto; contribuição é voluntária e financia operação coletiva.
+
+## 1) Resumo executivo
+
+Este plano substitui a lógica de monetização por planos fechados por um modelo de **Community-Sustained Infrastructure**:
+
+- não vender dados públicos,
+- não criar VIP/Premium,
+- não bloquear acesso por pagamento,
+- manter transparência financeira de custos e arrecadação.
+
+A camada de doação existe para manter a infraestrutura saudável e reduzir risco de abuso/custos imprevisíveis.
+
+## 2) Regras de produto (não-negociáveis)
+
+1. **Contribuição opcional (100%)**: ninguém é obrigado a pagar para usar o serviço.
+2. **Mesmo dado para todos**: a diferença é somente de cota/rate limit, nunca de conteúdo.
+3. **Sem paywall e sem venda de dados**.
+4. **Transparência pública**:
+   - custo mensal estimado de infra,
+   - valor arrecadado no ciclo,
+   - percentual da meta,
+   - destino de excedente.
+5. **Pós-meta**: quando a meta for atingida, novas doações são marcadas como `melhorias` ou `saldo_proximo_mes`.
+6. **Privacidade por padrão**: doação anônima default; reconhecimento público só por opt-in explícito.
+
+## 3) Como integrar APINow sem virar "plano premium"
+
+O APINow entra como **Tip Jar / Donation Processor**, não como provedor de paywall.
+
+### Fluxo
+
+1. Usuário vê no site a meta de infra (ex.: servidor + domínio + APIs).
+2. Usuário doa qualquer valor (ex.: R$ 10) via APINow.
+3. APINow envia webhook confirmando a doação.
+4. O sistema registra a contribuição e, opcionalmente, gera uma **chave de colaborador**.
+5. A chave concede **cota mais folgada** (uso justo), sem acesso a dados exclusivos.
+
+### Justiça do modelo
+
+- Quem não doa: continua usando com **cota comunitária**.
+- Quem doa: ajuda a financiar custo e recebe **cota colaboradora** para alto volume.
+- Resultado é o mesmo para ambos; muda apenas proteção de capacidade.
+
+## 4) Modelo de cotas (fair-use)
+
+## 4.1 Perfis
+
+- `community` (default): limite por IP/device/session (ex.: 50 consultas/dia).
+- `contributor` (chave ativa): limite por chave (ex.: 500 consultas/dia).
+
+> Os números são ilustrativos; calibrar por custo real de LLM e capacidade de infra.
+
+## 4.2 Garantias
+
+- **Paridade de dados**: resposta sem diferenciação semântica por perfil.
+- **Proteção anti-abuso**: rate limit + burst control + bloqueio por comportamento anômalo.
+- **Graceful degradation**: quando limite global estourar, manter health endpoint e resposta clara de indisponibilidade temporária.
+
+## 5) Arquitetura técnica (MVP)
+
+## 5.1 Tabelas
+
+```sql
+CREATE TABLE donation_campaigns (
+  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
+  name TEXT NOT NULL,
+  campaign_type TEXT NOT NULL CHECK (campaign_type IN ('infra','melhorias','saldo_proximo_mes')),
+  goal_amount_cents INTEGER NOT NULL,
+  raised_amount_cents INTEGER NOT NULL DEFAULT 0,
+  currency TEXT NOT NULL DEFAULT 'BRL',
+  status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','goal_reached','closed')),
+  starts_at TIMESTAMPTZ,
+  ends_at TIMESTAMPTZ,
+  created_at TIMESTAMPTZ DEFAULT now()
+);
+
+CREATE TABLE donations (
+  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
+  campaign_id UUID REFERENCES donation_campaigns(id),
+  user_id UUID REFERENCES auth.users(id), -- nullable para anônimo
+  amount_cents INTEGER NOT NULL CHECK (amount_cents > 0),
+  currency TEXT NOT NULL DEFAULT 'BRL',
+  donor_name TEXT,
+  is_anonymous BOOLEAN NOT NULL DEFAULT true,
+  message TEXT,
+  provider TEXT NOT NULL, -- ex: 'apinow'
+  external_payment_id TEXT NOT NULL,
+  status TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending','confirmed','failed')),
+  created_at TIMESTAMPTZ DEFAULT now(),
+  confirmed_at TIMESTAMPTZ,
+  UNIQUE (provider, external_payment_id)
+);
+
+CREATE TABLE contributor_keys (
+  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
+  donation_id UUID REFERENCES donations(id),
+  key_hash TEXT NOT NULL UNIQUE,
+  tier TEXT NOT NULL DEFAULT 'contributor' CHECK (tier IN ('contributor')),
+  daily_limit INTEGER NOT NULL,
+  status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','revoked','expired')),
+  expires_at TIMESTAMPTZ,
+  created_at TIMESTAMPTZ DEFAULT now()
+);
+```
+
+## 5.2 Endpoints sugeridos
+
+- `GET /api/donations/summary` (público)
+  - campanha ativa, meta, arrecadado, percentual, status, excedente.
+- `POST /api/donations/checkout` (público/autenticado)
+  - cria cobrança com valor livre (`min_amount_cents` opcional; ex.: >= 100).
+- `POST /api/webhooks/donations` (APINow)
+  - valida assinatura, confirma pagamento, faz upsert idempotente e atualiza agregado.
+- `GET /api/v1/infra/status` (público)
+  - custo estimado mensal, arrecadado, gap, status operacional.
+- `GET /api/v1/keys/me` (autenticado)
+  - retorna estado da chave colaboradora (sem exibir hash).
+
+## 5.3 Segurança e confiabilidade
+
+- Verificação de assinatura do webhook.
+- Idempotência por `(provider, external_payment_id)`.
+- Rate limit em `checkout` e endpoints sensíveis.
+- Armazenar só hash da chave colaboradora (nunca plaintext).
+- Auditoria de transições de status (`pending` → `confirmed`).
+- Logs sem dados pessoais sensíveis.
+

+- **Sim** haverá transparência contínua de custo e arrecadação.
